# Курсовая работа по ООП.
## Это что-то типа приложения по поиску и фильтрации вакансий с API HH.ru
    Здесь как то не потребовались никакие ключи API, поэтому нет никаких
    .env. Если брать другие сервисы, не ХХ, то можно добавить в классе
    HeadHunterAPI параметр запроса api_key и сохранить его в енв.
    Но придётся чуть переделать тесты с моками, иначе они будут падать.
### Модуль parser
    Здесь расположен абстрактный класс VacancyAPI для работы с API.
    Класс родительский.
### Модуль class_hh
    Здесь расположен класс HeadHunterAPI, который наследуется от родительского
    класса VacancyAPI. Класс инициализирует обьекты класса для отправки запросов в
    API и собирает ответ в формат json.
### Модуль vacancy_storage
    Здесь расположен абстрактный класс VacancyStorage.
    Класс родительмкий и снабжен методами для получения вакансии из файла,
    сохранения в файл и удаления из файла. Методы, соответсвенно, переопределяются
    в наследуемых классах.
### Модуль vacancy
    Класс ничего ни от кого не наследует, потому что нафиг не надо,
    а обрабатывает данные из хранилища.
    Здесь расположен класс Vacancy для обработки полученных вакансий и
    преобразования полученных результатов в словарь для сохранения,
    а так же преобразования словарей с вакансиями в объекты 
    класса Vacancy.
    Внутри класса определены методы, в том числе магические, для валидации 
    данных о зарплате, вычисления средней зарплаты, магические сравнения
    средней зарплаты по вакансиям и магический метод представления
    в строковом виде.
### Модуль json_vacancy_storage
    Здесь расположен класс JSONSaver, наследуемый от класса VacancyStorage.
    Класс работает файлом, в котором сохраняются обработаные данные
    по вакансиям.
    Внутри класса определяются методы для чтения данных из файла, записи в файл,
    добавления вакансии, удаления конкретной вакансии и удаления всех вакансий.
### Модуль vacancy_manager
    Здесь расположены функции (не классы) для обработки данных по вакансиям,
    записанным в файл.
    Тут есть фильтр по ключевому слову, по диапазону зарплат, сортировка по величине,
    получение топ N самых жирных вакансий (N берётся из пользовательского ввода),
    и, соответственно, вывода вакансий на монитор. Так то хорошо было бы воткнуть 
    для вывода миксин... Но я чёт не понял, как и куда его пристроить.
### Модуль data
    Здесь как раз и есть файл, куда сохраняются обработанные данные по вакансиям.
    Путь до него прописывается при инициализации в классе JSONSaver. И если, вдруг,
    такой путь не находится, то создаётся заново.
